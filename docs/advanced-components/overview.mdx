## App Extensions are separate processes

This is the biggest mental shift. Your share extension is *not* your app —
it's a sibling target with its own bundle, sandbox, and binary, sharing data
through an App Group container.

```swift
let groupID = "group.com.example.shop"
let url = FileManager.default
    .containerURL(forSecurityApplicationGroupIdentifier: groupID)?
    .appendingPathComponent("draft.json")
```

Use `App Groups` to share files, `UserDefaults(suiteName:)` for KV,
and the Keychain access group for secrets.

## WidgetKit timelines

```swift
struct ExpenseWidgetProvider: TimelineProvider {
    func getTimeline(in context: Context, completion: @escaping (Timeline<Entry>) -> Void) {
        let entries = (0..<6).map { offset in
            Entry(date: .now.addingTimeInterval(TimeInterval(offset * 3600)),
                  total: ExpenseStore.shared.todayTotal())
        }
        completion(Timeline(entries: entries, policy: .atEnd))
    }
}
```

Widgets are *not* live — you precompute future snapshots. iOS picks the right
entry to render at any given time.

## Key takeaways

- **Extensions are separate processes** — design data sharing via App Groups from day one.
- **Background tasks are unpredictable** — design idempotent work and don't promise schedules to users.
- **Widgets are timeline-driven**, not push-driven — recompute and refresh policies matter.
- **Permission strings live in Info.plist** — every microphone, camera, location, contacts use requires an `NS*UsageDescription`.
