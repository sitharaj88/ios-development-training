## Swift Concurrency — the full picture

<Diagram
  title="Swift Concurrency primitives"

>
  <SwiftConcurrencyDiagram />

```swift
actor RateLimiter {
    private var lastFire: Date = .distantPast
    private let interval: TimeInterval

    init(perSecond: Double) { interval = 1.0 / perSecond }

    func fire() async {
        let elapsed = Date().timeIntervalSince(lastFire)
        if elapsed < interval {
            try? await Task.sleep(for: .seconds(interval - elapsed))
        }
        lastFire = Date()
    }
}
```

Actors serialize access to mutable state; the compiler refuses to let
non-Sendable values cross actor boundaries.

## On-device ML with Core ML

```swift
import CoreML
import Vision

let model = try VNCoreMLModel(for: PetClassifier(configuration: .init()).model)
let request = VNCoreMLRequest(model: model) { req, _ in
    if let top = (req.results as? [VNClassificationObservation])?.first {
        print("\(top.identifier): \(top.confidence)")
    }
}
try VNImageRequestHandler(cgImage: image).perform([request])
```

Core ML runs on the Neural Engine — fast, private, no network required.

## visionOS at a glance

```swift
@main
struct GalaxyApp: App {
    var body: some Scene {
        WindowGroup { ContentView() }              // 2D window
        ImmersiveSpace(id: "Galaxy") {
            RealityView { content in
                content.add(galaxyEntity())        // 3D scene
            }
        }
    }
}
```

SwiftUI extends naturally into 3D — windows become `WindowGroup`s,
3D scenes are `ImmersiveSpace`s, and `RealityView` is the spatial canvas.

## Accessibility — non-optional

<Callout kind="warning" title="Accessibility is a launch blocker, not a polish task">
Apple App Store reviewers actively test VoiceOver. Apps with critical buttons
that don't speak get rejected. Build accessibility in from day one.
</Callout>

```swift
Image("trash")
    .accessibilityLabel("Delete item")
    .accessibilityAddTraits(.isButton)
    .accessibilityHint("Removes this from your cart")
```

Use Dynamic Type sizes, support landscape and split view, and test with
VoiceOver enabled (Settings → Accessibility → VoiceOver).

## Key takeaways

- **Actors + Sendable** are the future of safe shared state — the compiler enforces it.
- **Core ML runs on the Neural Engine** — embarrassingly fast, fully private.
- **visionOS is SwiftUI + RealityKit** — your existing skills extend.
- **Accessibility is shipped, not added later** — VoiceOver, Dynamic Type, and contrast from the first commit.
