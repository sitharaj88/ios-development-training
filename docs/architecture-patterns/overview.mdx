## MVVM with @Observable

```swift
@Observable
final class ProductListViewModel {
    enum State { case idle, loading, loaded([Product]), failed(String) }
    private(set) var state: State = .idle

    private let useCase: FetchProductsUseCase

    init(useCase: FetchProductsUseCase) {
        self.useCase = useCase
    }

    @MainActor
    func load() async {
        state = .loading
        do { state = .loaded(try await useCase.execute()) }
        catch { state = .failed(error.localizedDescription) }
    }
}

struct ProductListView: View {
    @State private var vm: ProductListViewModel
    var body: some View {
        Group {
            switch vm.state {
            case .idle, .loading: ProgressView()
            case .loaded(let products): List(products) { ProductRow($0) }
            case .failed(let msg): ErrorView(message: msg) { Task { await vm.load() } }
            }
        }
        .task { await vm.load() }
    }
}
```

Notice the view model has *no SwiftUI imports*. It's a plain Swift class
that's trivially testable.

## Repository protocols isolate the source

```swift
// Domain — pure Swift, no framework dependencies
protocol ProductRepository {
    func fetchAll() async throws -> [Product]
}

struct FetchProductsUseCase {
    let repo: ProductRepository
    func execute() async throws -> [Product] { try await repo.fetchAll() }
}

// Data — knows about URLSession, SwiftData, Firebase
final class ProductRepositoryImpl: ProductRepository {
    let api: ProductAPI
    let cache: ProductCache
    func fetchAll() async throws -> [Product] {
        if let cached = try? cache.load(), !cached.isEmpty { return cached }
        let fresh = try await api.products()
        try cache.save(fresh)
        return fresh
    }
}
```

The view model only knows the `ProductRepository` protocol — swap implementations
freely (test fakes, mock servers, offline mode).

## When to consider TCA

[The Composable Architecture](https://github.com/pointfreeco/swift-composable-architecture)
shines when your app is heavily state-driven (financial dashboards, IDE-like
tools, multi-step flows). It enforces unidirectional data flow,
exhaustive test coverage, and great time-travel debugging — at the cost of
boilerplate. Reach for it when MVVM stops scaling, not before.

## Key takeaways

- **Default to MVVM** with `@Observable` and `async`/`await`. It covers 80% of apps.
- **Keep Domain pure** — no SwiftUI, no URLSession, no SwiftData. Pure Swift only.
- **Repository protocols** are the seam where production code meets test fakes.
- **Coordinators** are not always necessary; reach for them when navigation rules cross features.
