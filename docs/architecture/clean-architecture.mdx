Clean Architecture (Robert C. Martin) and Hexagonal Architecture (Alistair Cockburn)
are the same idea: **inner layers know nothing of outer layers**. This document
shows how the principle maps to a real Swift project.

## The dependency rule

> Source code dependencies point inward, toward higher-level policy.

In an iOS app this means:

- `ProductListView` (SwiftUI) depends on `ProductListViewModel`.
- `ProductListViewModel` depends on `FetchProductsUseCase`.
- `FetchProductsUseCase` depends on the `ProductRepository` *protocol*.
- `ProductRepositoryImpl` depends on `URLSession`, `SwiftData`, etc.

Domain types (`Product`, `FetchProductsUseCase`, `ProductRepository`) live in
a Swift Package with **zero non-stdlib imports**. That is the litmus test.

## Use Cases — single-purpose actions

```swift
// Domain
struct CheckoutUseCase {
    let cartRepo: CartRepository
    let payment: PaymentGateway
    let orders:  OrderRepository

    func execute(method: PaymentMethod) async throws -> Order {
        let cart = try await cartRepo.current()
        try await payment.charge(amount: cart.total, method: method)
        let order = Order(items: cart.items, total: cart.total, placedAt: .now)
        try await orders.save(order)
        try await cartRepo.clear()
        return order
    }
}
```

The use case orchestrates protocols. Every dependency is injectable. There's
no SwiftUI, no URLSession, no SwiftData visible.

## Repositories isolate sources

```swift
// Domain
protocol ProductRepository {
    func fetchAll() async throws -> [Product]
    func find(id: Product.ID) async throws -> Product?
}

// Data
final class ProductRepositoryImpl: ProductRepository {
    private let api: ProductAPI       // remote
    private let cache: SwiftDataCache // local

    func fetchAll() async throws -> [Product] {
        if let local = try? cache.products(), !local.isEmpty { return local }
        let remote = try await api.products()
        try cache.replace(with: remote)
        return remote
    }

    func find(id: Product.ID) async throws -> Product? {
        try await cache.find(id: id) ?? api.product(id)
    }
}
```

The repository owns caching policy, mapping, and failover. Use cases don't care.

## Mapping vs domain leaks

DTOs (`ProductDTO` from JSON) are *not* domain entities (`Product`).
Map at the repository boundary so that snake_case keys, missing fields, or
backwards-incompatible API changes never leak past the data layer.

```swift
extension ProductDTO {
    func toDomain() -> Product {
        Product(id: .init(uuidString: id) ?? .init(),
                name: name,
                price: Decimal(string: price) ?? 0)
    }
}
```

<Callout kind="warning" title="Codable on domain types is a smell">
If `Product` is `Codable`, you've coupled your domain to your wire format.
The day a backend rename happens you'll need to migrate every screen. Map
at the boundary instead.
</Callout>
