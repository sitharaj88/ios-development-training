One-page reference for SwiftUI idioms. Targets iOS 17+ (Observation, new
navigation, `#Preview`). Where legacy syntax matters, it's called out
explicitly.

## Minimum view

```swift
import SwiftUI

struct Hello: View {
    var body: some View {
        Text("Hello, world")
            .font(.title)
            .padding()
    }
}

#Preview {
    Hello()
}
```

## State property wrappers (iOS 17+)

| Wrapper | Use when… |
| --- | --- |
| `@State` | Value-type view-local state |
| `@Binding` | Two-way ref to state owned by parent |
| `@Bindable` | Derive `$` bindings from an `@Observable` class |
| `@Environment` | Read a system or injected value |
| `@Observable` | Decorate a reference-type model for SwiftUI tracking |
| `@AppStorage` | `UserDefaults`-backed state |
| `@SceneStorage` | Per-scene state, restored on relaunch |
| `@FocusState` | Track focus target for text fields |

Legacy (pre-iOS 17) equivalents: `@StateObject`, `@ObservedObject`,
`@EnvironmentObject`, `ObservableObject`.

## Observation (iOS 17+)

```swift
@Observable
final class CartViewModel {
    var items: [Item] = []
    var isLoading = false
    func add(_ item: Item) { items.append(item) }
}

struct CartView: View {
    @Bindable var vm: CartViewModel

    var body: some View {
        List(vm.items) { Text($0.name) }
    }
}
```

## Navigation (iOS 16+)

```swift
NavigationStack(path: $router.path) {
    HomeView()
        .navigationDestination(for: Route.self) { route in
            switch route {
            case .profile(let id): ProfileView(id: id)
            case .settings:         SettingsView()
            }
        }
}

// Programmatic push
router.path.append(Route.profile(id: user.id))
```

## Modifiers — ordering matters

```swift
Text("Hi")
    .padding()            // 1. pad the text
    .background(.blue)    // 2. fill behind the padded rectangle
    .clipShape(.capsule)  // 3. mask to capsule

Text("Hi")
    .background(.blue)    // 1. fill behind text-sized rectangle
    .padding()            // 2. pad the filled capsule-less shape
```

## Lifecycle hooks

```swift
.task { await vm.load() }                   // iOS 15+, auto-cancels
.task(id: user.id) { await vm.reload() }    // re-run on value change
.onAppear { analytics.viewed() }
.onChange(of: query) { old, new in vm.search(new) }
.refreshable { await vm.reload() }
```

## Environment values

Inject your own:

```swift
struct APIClientKey: EnvironmentKey {
    static let defaultValue: APIClient = .live
}
extension EnvironmentValues {
    var apiClient: APIClient {
        get { self[APIClientKey.self] }
        set { self[APIClientKey.self] = newValue }
    }
}

ContentView().environment(\.apiClient, .live)
```

## Common pitfalls

- **Putting business logic in `body`** — `body` is re-evaluated often; keep
  it pure. Move work into the view model.
- **`@State` on reference types** — use `@Observable` + `@Bindable` or
  `@StateObject`.
- **Excessive modifier re-application** — prefer conditional *values* over
  conditional *modifiers* to avoid SwiftUI's "view identity" gotchas.
- **Heavy `onAppear` work** — use `.task { ... }` so it auto-cancels on
  disappearance.
