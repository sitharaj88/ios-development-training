## SwiftData in 30 lines

```swift
@Model
final class Expense {
    var amount: Decimal
    var category: String
    var date: Date
    init(amount: Decimal, category: String, date: Date = .now) {
        self.amount = amount
        self.category = category
        self.date = date
    }
}

@main
struct WalletApp: App {
    var body: some Scene {
        WindowGroup { ExpenseList() }
            .modelContainer(for: Expense.self)
    }
}

struct ExpenseList: View {
    @Query(sort: \.date, order: .reverse) private var expenses: [Expense]
    @Environment(\.modelContext) private var context

    var body: some View {
        List(expenses) { Text("\($0.category): \($0.amount)") }
        Button("Add") {
            context.insert(Expense(amount: 12.5, category: "Coffee"))
        }
    }
}
```

`@Query` re-runs and updates the view automatically when the model context changes.

## Keychain — the right way

```swift
enum KeychainStore {
    static func save(_ token: String, account: String) throws {
        let data = Data(token.utf8)
        let q: [String: Any] = [
            kSecClass as String       : kSecClassGenericPassword,
            kSecAttrAccount as String : account,
            kSecValueData as String   : data,
        ]
        SecItemDelete(q as CFDictionary)
        let status = SecItemAdd(q as CFDictionary, nil)
        guard status == errSecSuccess else { throw KeychainError.save(status) }
    }
}
```

<Callout kind="warning" title="Never store secrets in UserDefaults">
UserDefaults files are plaintext plists inside the sandbox. Anyone with a backup
or a jailbroken device can read them. Tokens, passwords, biometric data, and
anything compliance-relevant goes in Keychain.
</Callout>

## CloudKit sync (free tier!)

`NSPersistentCloudKitContainer` (Core Data) and SwiftData with
`ModelConfiguration(cloudKitDatabase: .private(...))` add iCloud sync with
conflict resolution out of the box. Limits: ~10GB private, 100MB asset size,
and your users need iCloud signed in.

## Key takeaways

- **Pick storage by data shape**: KV → UserDefaults · Secrets → Keychain · Object graph → SwiftData/Core Data · Sync → CloudKit.
- **SwiftData is the new default** for iOS 17+. Core Data remains relevant for back-deployment and advanced features.
- **`@Query` is reactive** — pair it with `@Observable` view models for clean data flow.
- **Migrations are not optional** — design schema versions from day one.
