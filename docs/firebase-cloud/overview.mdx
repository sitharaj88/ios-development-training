## Sign in with Apple — non-negotiable

App Store Review Guideline 4.8 requires Sign in with Apple if you offer any
third-party social login. Implement it first.

```swift
import AuthenticationServices

struct SignInWithAppleButton: UIViewRepresentable {
    let onResult: (Result<ASAuthorizationAppleIDCredential, Error>) -> Void
    // ...wires ASAuthorizationController + ASAuthorizationAppleIDProvider
}
```

Pair with Firebase Auth's `OAuthProvider(providerID: "apple.com")` to get a
Firebase user back, with the Apple identity token verified server-side.

## Firestore listeners are the magic

```swift
db.collection("orders")
  .whereField("userId", isEqualTo: uid)
  .addSnapshotListener { snap, error in
      let orders = snap?.documents.compactMap { try? $0.data(as: Order.self) } ?? []
      Task { @MainActor in self.orders = orders }
  }
```

Every device watching that query gets pushed updates within ~100ms.
Combine with offline persistence (`Firestore.firestore().settings.isPersistenceEnabled = true`)
and your app feels instant even on the subway.

<Callout kind="warning" title="Security rules are not optional">
A Firestore project with `allow read, write: if true;` is a public database.
Write your rules and test them with the emulator suite *before* shipping.
</Callout>

## Crashlytics that's actually useful

Two non-negotiables:

1. **Upload dSYMs** on every release — without them stack traces are hex addresses.
2. **Set user identifiers** so you can correlate crashes to support tickets:

```swift
Crashlytics.crashlytics().setUserID(currentUser.id)
Crashlytics.crashlytics().setCustomValue(appBuild, forKey: "build")
```

Use the run-script build phase that Firebase ships, or upload via Fastlane in CI.

## Key takeaways

- **Sign in with Apple is mandatory** if you offer any third-party login.
- **Firestore listeners drive reactive UIs cheaply** — embrace them.
- **Security rules are part of the app** — write them, test them, deploy them with the binary.
- **Crash reports without dSYMs are useless** — automate dSYM upload in CI.
