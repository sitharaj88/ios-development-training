## Why UIKit still matters

You will write SwiftUI for almost everything new. But:

- Most enterprise codebases are 80%+ UIKit and will be for years.
- Several SwiftUI APIs have UIKit-only equivalents (custom transitions, advanced
  text input, complex collection layouts).
- Debugging weird SwiftUI behavior often requires inspecting the underlying
  `UIHostingController` and view hierarchy.
- Interview questions still lean heavily on UIKit fundamentals.

<Callout kind="tip" title="The right mindset">
Treat UIKit as the *runtime model* of iOS UI. Even when you live in SwiftUI,
knowing `viewDidLoad`, the responder chain, and the run loop pays compounding
dividends.
</Callout>

```swift
@main
struct ShopApp: App {
    @Environment(\.scenePhase) private var scenePhase

    var body: some Scene {
        WindowGroup {
            RootView()
        }
        .onChange(of: scenePhase) { _, phase in
            if phase == .background { saveDraft() }
        }
    }
}
```

## The view controller lifecycle (still essential)

```swift
class ProductListVC: UIViewController {
    override func viewDidLoad() {
        super.viewDidLoad()
        // Outlets connected; one-time setup here.
    }
    override func viewWillAppear(_ animated: Bool) {
        super.viewWillAppear(animated)
        // Refresh data; about to be visible.
    }
    override func viewDidAppear(_ animated: Bool) {
        super.viewDidAppear(animated)
        // Start animations / analytics.
    }
}
```

Every SwiftUI view that's pushed via UIKit eventually lives inside a
`UIHostingController` whose lifecycle drives `onAppear`/`task`/`onDisappear`
under the hood.

## Auto Layout in 30 seconds

```swift
NSLayoutConstraint.activate([
    button.centerXAnchor.constraint(equalTo: view.centerXAnchor),
    button.bottomAnchor.constraint(equalTo: view.safeAreaLayoutGuide.bottomAnchor, constant: -16),
    button.heightAnchor.constraint(equalToConstant: 48),
])
```

When constraints conflict, Xcode logs `UIViewAlertForUnsatisfiableConstraints`
and synthesizes an answer. We cover debugging by priority and `.required` vs
`.defaultHigh` in detail.

## Key takeaways

- The **AppDelegate / SceneDelegate / SwiftUI App** picture maps to the same OS lifecycle.
- **`UIViewController` lifecycle methods fire in a deterministic order** — memorize them.
- **Auto Layout is constraint solving** — priorities and `safeAreaLayoutGuide` are non-negotiable on modern iPhones.
- **Sandbox boundaries** dictate where you can write files, what URLs you can hit, and what user data requires permission strings.
