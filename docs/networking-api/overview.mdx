## URLSession with async/await

```swift
struct ProductAPI {
    let base = URL(string: "https://api.example.com")!
    let session: URLSession = .shared
    let decoder = JSONDecoder()

    func products() async throws -> [Product] {
        var req = URLRequest(url: base.appending(path: "products"))
        req.setValue("application/json", forHTTPHeaderField: "Accept")
        let (data, response) = try await session.data(for: req)
        try Self.assertOK(response)
        return try decoder.decode([Product].self, from: data)
    }

    static func assertOK(_ response: URLResponse) throws {
        guard let http = response as? HTTPURLResponse, (200..<300).contains(http.statusCode)
        else { throw APIError.badStatus }
    }
}
```

## Codable that survives real APIs

```swift
struct Product: Codable, Identifiable {
    let id: UUID
    let name: String
    let price: Decimal
    let imageURL: URL?

    enum CodingKeys: String, CodingKey {
        case id, name, price
        case imageURL = "image_url"
    }
}
```

Use `JSONDecoder.keyDecodingStrategy = .convertFromSnakeCase` to skip
manual key mapping; use `dateDecodingStrategy = .iso8601` for sane dates.

## Token refresh that doesn't deadlock

The naïve approach (intercept 401, refresh, retry) breaks when 50 concurrent
requests all 401 at once and each tries to refresh independently. The pattern
that works:

```swift
actor TokenRefresher {
    private var refreshTask: Task<String, Error>?

    func validToken() async throws -> String {
        if let token = current, !token.isExpired { return token.value }
        if let task = refreshTask { return try await task.value }
        let task = Task { try await self.performRefresh() }
        refreshTask = task
        defer { refreshTask = nil }
        return try await task.value
    }
}
```

`actor` serializes access; the in-flight `refreshTask` deduplicates concurrent
callers automatically.

## WebSocket and AsyncSequence

```swift
let task = URLSession.shared.webSocketTask(with: wsURL)
task.resume()

Task {
    for try await message in task.messages {
        switch message {
        case .string(let s): print("text: \(s)")
        case .data(let d):   print("data: \(d.count)B")
        @unknown default:    break
        }
    }
}
```

`task.messages` is an `AsyncSequence` — `for try await` is the modern API.

## Key takeaways

- **`URLSession` + `async`/`await` + `Codable`** is the modern default — no client library required for most apps.
- **Centralize HTTP** in a single API service per backend; never call `URLSession` from a view model.
- **Refresh-token flows must be serialized** — use an `actor` or the equivalent.
- **Cache aggressively** — image cache for assets, `URLCache` for HTTP, your own cache for parsed objects.
