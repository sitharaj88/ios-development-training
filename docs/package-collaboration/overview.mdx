## SPM is the default

```swift
// swift-tools-version:5.10
import PackageDescription

let package = Package(
    name: "FeatureCart",
    platforms: [.iOS(.v16)],
    products: [.library(name: "FeatureCart", targets: ["FeatureCart"])],
    dependencies: [
        .package(url: "https://github.com/apple/swift-collections", from: "1.0.0"),
    ],
    targets: [
        .target(name: "FeatureCart", dependencies: [.product(name: "Collections", package: "swift-collections")]),
        .testTarget(name: "FeatureCartTests", dependencies: ["FeatureCart"]),
    ]
)
```

For modular monoliths, split each feature into its own SPM target. Build times
go down, test isolation goes up, and the dependency arrows become explicit.

## Git workflow that scales

```
main           ─── (always deployable)
  └─ feature/  ─── short-lived branches, squash-merged via PR
  └─ release/  ─── only for app version cuts
  └─ hotfix/   ─── urgent fixes, cherry-picked back to main
```

Pair with **Conventional Commits** (`feat:`, `fix:`, `chore:`) so changelogs
generate themselves and CI can decide version bumps.

<Callout kind="tip" title="Trunk-based for product teams, GitFlow for SDK teams">
Trunk-based development with feature flags is faster for product apps. GitFlow
makes more sense when you ship versioned libraries with multiple supported
release lines.
</Callout>

## CI: Xcode Cloud vs GitHub Actions

```yaml
# .github/workflows/ios.yml
name: iOS
on: [push, pull_request]
jobs:
  test:
    runs-on: macos-14
    steps:
      - uses: actions/checkout@v4
      - run: xcodebuild -scheme MyApp -sdk iphonesimulator -destination 'platform=iOS Simulator,name=iPhone 15'
              test | xcpretty
```

Xcode Cloud requires a paid developer account but ties beautifully into App
Store Connect (TestFlight uploads happen automatically). GitHub Actions is
cheaper and more flexible for non-Apple workflows.

## Fastlane lanes

```ruby
# Fastfile
default_platform :ios

lane :beta do
  match(type: "appstore")
  build_app(scheme: "MyApp")
  upload_to_testflight(skip_waiting_for_build_processing: true)
end

lane :release do
  match(type: "appstore")
  build_app(scheme: "MyApp")
  upload_to_app_store(submit_for_review: true, automatic_release: true)
end
```

Two commands (`fastlane beta`, `fastlane release`) replace dozens of clicks.

## Key takeaways

- **SPM is the default**. Reach for CocoaPods only when an SDK forces you.
- **Modularize features into Swift Packages** — build times and test isolation both improve.
- **Conventional Commits + squash merges** keep history clean and changelogs automatic.
- **Automate releases with Fastlane or Xcode Cloud** — two-command shipping is the bar.
